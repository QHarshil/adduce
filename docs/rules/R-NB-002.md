# R-NB-002 — Committed outputs likely stale relative to the code

**Category:** Notebooks  
**Weight:** 2

## Why it matters

Outputs from a different execution order are results the current code may not produce.

## Suppressing

Inline, on the reported line: `# adduce: ignore=R-NB-002`  
Project-wide, in `adduce.toml` or `[tool.adduce]`: `ignore = ["R-NB-002"]`

Suppressed findings still appear in reports, marked as ignored.
