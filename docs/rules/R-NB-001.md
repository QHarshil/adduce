# R-NB-001 — Notebooks executed in linear order

**Category:** Notebooks  
**Weight:** 3

## Why it matters

Non-monotonic execution counts mean the saved outputs were produced in a different order than the code reads — rerunning top-to-bottom may not reproduce them.

## Suppressing

Inline, on the reported line: `# adduce: ignore=R-NB-001`  
Project-wide, in `adduce.toml` or `[tool.adduce]`: `ignore = ["R-NB-001"]`

Suppressed findings still appear in reports, marked as ignored.
